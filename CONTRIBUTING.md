# 🤝 Contributing to DataForge CLI

First of all, thank you for your interest in contributing to **DataForge CLI**.

Whether you're fixing a typo, reporting a bug, improving the documentation, or proposing a new feature, every contribution helps make the project better.

Our goal is to build a toolkit that is **Local-first, Fast and Professional**, while keeping the codebase clean, maintainable and easy to understand.

---

# 🐞 Reporting Bugs

Found a bug?

Before opening an Issue, please make sure to include:

- Operating System (Windows, macOS or Linux)
- Python version
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or logs (if applicable)

The more information you provide, the easier it is to reproduce and fix the problem.

---

# 💡 Suggesting New Features

Have an idea?

Feature requests are always welcome.

Please explain:

- What problem does it solve?
- Why would it benefit other developers?
- How do you imagine it working?

Not every suggestion will be implemented, but every idea will be reviewed.

---

# 💻 Coding Standards

To keep the project consistent, please follow these guidelines:

- Follow PEP 8 whenever possible.
- Write clean and readable code.
- Keep functions focused on a single responsibility.
- Use meaningful variable and function names.
- Document public functions when appropriate.

---

# 🔀 Pull Requests

Before submitting a Pull Request:

- Make sure your code works correctly.
- Test your changes.
- Update the documentation if necessary.
- Keep each Pull Request focused on a single improvement.

Small, focused Pull Requests are much easier to review than large ones.

---

# 📌 Project Direction

DataForge CLI is maintained by **RGS Labs™**.

Everyone is welcome to contribute ideas, bug fixes and improvements.

However, **RGS Labs™ reserves the final decision** on what gets merged into the project in order to maintain code quality, architectural consistency and the long-term vision of DataForge CLI.

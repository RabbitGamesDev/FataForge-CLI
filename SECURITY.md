# 🛡️ Security Policy

Thank you for helping keep **DataForge CLI** secure.

Security is one of the core principles of this project. DataForge CLI follows a **Local-first** philosophy, meaning your source code, configuration files and sensitive information remain under your control whenever possible.

If you discover a security vulnerability, we appreciate your responsible disclosure.

---

# 🔒 Security Philosophy

DataForge CLI has been designed with security and privacy in mind.

Current security principles include:

- API Keys are stored locally.
- Sensitive files are excluded from version control.
- DataForge never modifies your source code.
- Reports are generated separately from your project.
- The project follows a Local-first approach whenever possible.

Security improvements are continuously reviewed and implemented as the project evolves.

---

# 🚨 Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

Instead, send a detailed report to:

📧 **rgslabs.contact@gmail.com**

Please include as much information as possible:

- DataForge CLI version
- Operating System
- Python version
- Description of the vulnerability
- Steps to reproduce
- Proof of concept (if available)
- Screenshots or logs (optional)

The more information you provide, the faster the issue can be investigated.

---

# ⏱️ Response Time

RGS Labs™ aims to acknowledge all security reports within **72 hours**.

After confirming the issue, we will investigate it and work on an appropriate fix as soon as possible.

Please understand that resolution time depends on the complexity and impact of the reported vulnerability.

---

# ✅ Supported Versions

Only the latest stable release of DataForge CLI is actively supported with security updates.

Older versions may not receive fixes.

| Version | Supported |
|----------|-----------|
| Latest Release | ✅ |
| Older Releases | ❌ |

# ❤️ Responsible Disclosure

We kindly ask security researchers and contributors to:

- Give us reasonable time to investigate and fix the issue.
- Avoid publicly disclosing vulnerabilities before a fix is available.
- Report vulnerabilities responsibly and in good faith.

Your help makes DataForge CLI safer for everyone.
